标签预测API文档
接口路径
POST /tags/predict

描述
该API接口接收文本内容，并根据指定的检查标签（p0、p1、p2_1或p2_2）返回预测的标签及其从父节点到叶节点的层级路径。它通过在单个请求中接受多个文本内容来支持批量操作。

请求头
Content-Type: application/json
请求体
字段	类型	描述
contents	Array	字符串数组，每个字符串代表一段要分析的文本内容。
id	String	请求的唯一标识符。
source_type	String	内容来源的类型（例如，news、article）。
check_tag	String	指定用于预测的模型。可以是p0、p1、p2_1或p2_2。对于p2，将应用p2_1和p2_2模型。
示例
json
Copy code
{
  "contents": [
    "文本内容1",
    "文本内容2"
  ],
  "id": "123456789",
  "source_type": "news",
  "check_tag": "p0"
}
响应体
字段	类型	描述
id	String	回显请求中的唯一标识符。
source_type	String	回显请求中的内容来源类型。
check_tag	String	回显请求中指定的预测模型。
label_list	Array	对象数组，每个对象包含content（输入文本）、labels（内容的预测标签）和label_hierarchy_paths（预测标签的层级路径）。
示例
json
Copy code
{
  "id": "123456789",
  "source_type": "news",
  "check_tag": "p0",
  "label_list": [
    {
      "content": "文本内容1",
      "labels": ["标签1", "标签2"],
      "label_hierarchy_paths": ["父级 > 子级 > 标签1", "父级 > 子级 > 标签2"]
    },
    {
      "content": "文本内容2",
      "labels": ["标签3"],
      "label_hierarchy_paths": ["父级 > 子级 > 标签3"]
    }
  ]
}
错误响应
如果请求中缺少任何必需字段，API将响应400状态码，并提供详细的缺少字段错误信息。

示例
json
Copy code
{
  "error": "缺少必需字段：contents"
}
